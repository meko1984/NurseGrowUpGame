# 看護師道場-新人成長物語-

看護判断の学習ゲーム「看護師道場-新人成長物語-」の初期プロトタイプです。スマートフォンの縦画面を優先し、HTML / CSS / JavaScriptのみで作成しています。

## 今回実装した内容

- スマホ向けメイン画面
- ユニット病室マップ
- 4床表示（肺炎、心不全、消化管出血、空床）
- 4床のベッドサイドモニターと共通アラーム判定
- 各床の点滴棒と患者氏名・年齢表示
- タップ地点へのプレイヤー移動アニメーション
- ベッド・設備のオブジェクトタップ
- カルテ画面と患者一覧
- 患者別の指示・検査値・記録・申し送りメニュー
- 患者別のケア・注射指示書・内服処方
- 薬剤マスターと患者別処方を分離した内服データ構造
- `data/` 配下の患者マスター、ベッド割当、薬剤、内服処方、注射指示書JSON
- `quests.json`による4分類対応のクエスト管理と肺炎患者Lv1〜Lv10クエストツリー
- `player.json`によるプレイヤー状態、受注中・達成済みID、クエスト報酬EXP・レベルアップ処理
- `events.json`と、行動後の時刻に応じてイベントを確認する処理
- `care_master.json`と`patient_care_plans.json`によるケア項目・患者別ケア計画の分離
- 基準値に応じた検査値の高値・低値表示
- 4床分のセントラルモニター表示
- 注意アラーム表示
- アラーム確認と対象ベッドへの移動
- 点滴台など7設備の簡易画面
- 手洗い、物品棚、ゴミ箱、救急カートのミニゲーム入口画面
- 移動ごとにゲーム内時刻を5分進める処理
- Bed1のRR・SpO₂観察アクションとLv1〜Lv3の達成判定
- 受注可能・進行中・達成済み・未解放を確認できるクエスト一覧
- `localStorage`によるプレイヤーとクエスト進捗の保存（利用可能なブラウザのみ）
- PCでも利用できるレスポンシブ表示

## データ設計

JSONは、ExcelやGoogleスプレッドシートから変換しやすいよう、原則として1行相当のレコードを1オブジェクトにした配列で管理しています。`script.js`は読み込んだ元データからプレイ中の状態をコピーして使うため、将来は保存APIやデータベースへ置き換えられます。

- `data/quests.json`：`questType`、必要レベル、actionId、内包クエスト、報酬、調整用コメント
- `data/player.json`：レベル、EXP、体力、やる気などの初期値
- `data/events.json`：発生時刻、イベント種別、対象患者、変更値
- `data/care_master.json`：ケア項目、必要物品、重要度、ミニゲーム種別
- `data/patient_care_plans.json`：患者が選択したケアと理想ケアの紐付け

クエスト進捗とプレイヤー状態は`localStorage`へ保存します。イベント発火状態やケア実施状態の永続化は未実装です。外部DBへ移行する場合は、現在のランタイム状態と同じID構造で保存APIへ置き換える想定です。

## 肺炎クエスト

Bed1（`patientId: p001`）を対象にLv1〜Lv10を登録しています。Lv1〜Lv3はRR／SpO₂観察から達成、経験値獲得まで動作します。Lv4以降は`requiredActions`と`coversQuestIds`を先行定義し、プレイヤーレベルに応じて解放されます。

レベルアップ必要EXPはLv1から順に30、60、100、150です。Lv5以降は現在レベル×40を仮値として使います。クエスト分類は`questType`で「昇段試験」「看護クエスト」「特別クエスト」「プチクエスト」を扱えます。
