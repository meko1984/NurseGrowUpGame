# 看護師道場-新人成長物語-

看護判断の学習ゲーム「看護師道場-新人成長物語-」の初期プロトタイプです。スマートフォンの縦画面を優先し、HTML / CSS / JavaScriptのみで作成しています。

## 起動方法

JSONデータを読み込むため、プロジェクト直下をローカルWebサーバーで配信して開いてください。ビルドや外部ライブラリは不要です。

例：`python3 -m http.server 8000` を実行し、`http://localhost:8000` を開きます。

## GitHub Pagesで公開する

このリポジトリは、ルートに`index.html`、`style.css`、`script.js`、`data/`を置いたGitHub Pages対応構成です。JSONは`./data/ファイル名.json`で読み込むため、`https://ユーザー名.github.io/リポジトリ名/`のようなプロジェクト配下URLでも動作します。

1. このプロジェクト一式をGitHubリポジトリのデフォルトブランチへpushします。
2. GitHubのリポジトリ画面で **Settings → Pages** を開きます。
3. **Build and deployment** のSourceを **Deploy from a branch** にします。
4. Branchでデフォルトブランチ（通常は`main`）と`/(root)`を選び、**Save**を押します。
5. デプロイ完了後、Pages画面に表示される公開URLをスマートフォンで開きます。

更新後は反映まで数分かかる場合があります。JSONが404になる場合は、`data/`を含めてpushされていること、ファイル名の大文字・小文字が一致していること、公開元が`/(root)`であることを確認してください。

`.nojekyll`を含めているため、ファイルはJekyll変換を介さずそのまま配信されます。

## データ設計

JSONは、ExcelやGoogleスプレッドシートから変換しやすいよう、原則として1行相当のレコードを1オブジェクトにした配列で管理しています。`script.js`は読み込んだ元データからプレイ中の状態をコピーして使うため、将来は保存APIやデータベースへ置き換えられます。

- `data/quests.json`：クエスト定義、手順、報酬、調整用コメント
- `data/player.json`：レベル、EXP、体力、やる気などの初期値
- `data/events.json`：発生時刻、イベント種別、対象患者、変更値
- `data/care_master.json`：ケア項目、必要物品、重要度、ミニゲーム種別
- `data/patient_care_plans.json`：患者が選択したケアと理想ケアの紐付け

現在はブラウザを再読み込みすると初期値へ戻ります。保存機能を追加する際は、プレイヤー、クエスト進捗、イベント発火状態、ケア実施状態をランタイム状態から保存する想定です。
